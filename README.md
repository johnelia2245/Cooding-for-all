<h2>وصف المشروع
<h3>In today’s world, teaching children programming helps develop their imagination and strengthens their problem-solving skills. Coding is not just a technical skill; it allows kids to move from simply using technology to creating and shaping it. By learning how to communicate with computers, they build creativity, persistence, and the ability to turn their ideas into reality.

<h2>مسؤولية كل عضو:


| العضو | الدور | المهام |
|-------|------|--------|
| 1-فاطمه تامر محمد | Product & UX Lead | - صياغة فكرة المشروع وكتابة وصفه<br>- تحديد الأهداف والوظائف الأساسية<br>- البحث عن المستخدمين (استبيانات، مقابلات)<br>- إنشاء Personas وUser Flows<br>- متابعة وضوح الفكرة طوال المشروع |
| 2-فرح محمد حجازي | Wireframes & Interaction Designer | - تصميم المخططات الأولية (Wireframes)<br>- رسم تدفق المستخدم (User Flow)<br>- تجهيز نسخة أولية للتصميم للتقييم الداخلي<br>- المراجعة مع الفريق قبل UI |
| 3-جون ايليا رسمي | UI Designer | - تصميم واجهة المستخدم (High-Fidelity)<br>- اختيار الألوان، الخطوط، الأزرار، القوائم<br>- تنفيذ التصميم في Figma / Adobe XD<br>- التأكد من تطبيق Design System موحد |
| 4-فبرونيا سامح يوسف | UI Designer | - تصميم واجهة المستخدم (High-Fidelity)<br>- التعاون مع العضو 3 في تنفيذ التصميمات<br>- مراجعة التفاصيل واللمسات النهائية للواجهة |
| 5-لؤي محمد محمود | Usability & Documentation Lead | - إجراء اختبارات الاستخدام (Usability Testing)<br>- جمع الملاحظات وتحليلها<br>- توثيق المشروع بالكامل (Screenshots, Prototypes)<br>- تجهيز العرض النهائي للمدرس أو اللجنة |

> => ملاحظات:
> - كل عضو يشارك في مراجعة العمل المشترك لضمان اتساق التصميم مع أهداف المشروع.

> - المهام يمكن أن تتداخل قليلاً حسب الحاجة، خصوصًا بين UI وWireframes.
